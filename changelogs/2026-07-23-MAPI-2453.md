## What Changed

- OpenAPI source is now multi-file: path operations live under `spec/paths/<tag>/` and schemas under `spec/components/schemas/<tag>/` (shared schemas stay at the schemas root).
- Docs build bundles the split entry first, then injects code samples, then builds ReDoc — published output path remains `web_deploy/index.html`.
- `npm run inject` bundles to `.tmp/openapi.bundled.yaml` before injecting, so it works as a standalone script (the inject script itself remains injection-only).
- README documents the split layout; `securitySchemes` stay inline in `openapi.yaml` (no `securitySchemes/` directory).
- Create webhook template-parameter docs note that `$type` is deprecated; use `$format` instead.
- Dedicated-numbers assignments example uses concrete metadata values instead of a dangling `#/components/schemas/Metadata` `$ref`.

## Why

A single ~10k-line OpenAPI file was hard to maintain; splitting by service/tag makes updates safer without changing the published API docs.
